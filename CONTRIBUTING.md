
# Contributing to Orizon

## Branching
- `main`: stable
- `dev`: integration
- feature branches: `feat/<area>-<short-desc>`

## Commits
Use conventional commits: `feat:`, `fix:`, `docs:`, `chore:`, `test:`, `refactor:`.

## Pull Requests
- Link the Issue.
- Add screenshots for UI changes.
- Ensure tests pass (CI).
- At least 2 approvals.

## Code Style
- TypeScript preferred for API & Web.
- ESLint + Prettier enforced.
- Zod for runtime validation at API boundaries.
